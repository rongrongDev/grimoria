# Glossary — AI Evaluation Engineering

**Version:** 1.0.0 · **Date:** 2026-07-06. Single source of truth for terms used across this KB. Terms link to the doc that treats them in depth.

- **A/A test / A/A run** — Running the identical system (or identical outputs, for a pairwise judge) through the eval twice. Any difference is measurement noise or harness bias, by construction. Establishes the *noise band*; the cheapest instrument-health check that exists. → `principles/regression-testing-and-edd.md`, `principles/llm-as-judge.md`
- **Attack success rate (ASR)** — Fraction of adversarial probes that elicit a policy violation, reported per category with CI; meaningful as a trend on a frozen set, weak as an absolute. → `topics/adversarial-evaluation.md`
- **Baseline (pinned)** — A specific recorded run (system commit + eval-set version + judge hash) that comparisons are made against; distinct from "last week's number," which floats. → `principles/regression-testing-and-edd.md`
- **Calibration (of a judge)** — Validating LLM-judge verdicts against qualified-human labels on a stratified sample; the only evidence a judge measures your rubric. Measured with kappa. → `principles/llm-as-judge.md`
- **Confidence interval (CI)** — Range of plausible true values given sampling noise; a score without one is an anecdote. Wilson interval for pass rates; bootstrap for everything else. → `principles/statistical-rigor.md`
- **Contamination** — Eval items (or near-duplicates, or shared-provenance derivatives) present in training data, converting a generalization test into a memorization test. Channels: direct, near-duplicate, iteration leakage. → `principles/contamination-and-leakage.md`
- **Dev set vs. held-out set** — Dev: iterated against freely, and therefore progressively *burned* (overfit via engineer decisions). Held-out: scored rarely, never used to select between changes; the honest number. → `principles/contamination-and-leakage.md` §3
- **Eval-driven development (EDD)** — Treating model/prompt changes like code changes: no merge without eval evidence, tiered CI gates, versioned fixtures. → `principles/regression-testing-and-edd.md`
- **Flake band / noise band** — Measured run-to-run score variance under no change (from A/A runs); gate thresholds below it fire randomly and teach re-run-until-green. → `principles/regression-testing-and-edd.md`
- **Golden set** — The versioned collection of inputs + expected outputs/rubrics defining an eval. Its composition *is* the eval's meaning; its hygiene rules (immutability per version, provenance, strata) are non-negotiable. → `principles/eval-design.md`
- **Ground truth** — The reference answer/label an eval scores against; only as good as its provenance (who decided, with what expertise). Wrong ground truth penalizes correct behavior invisibly and forever. → `principles/eval-design.md`, `principles/human-evaluation.md`
- **Inter-rater reliability (IRR)** — Chance-corrected agreement between annotators: Cohen's kappa (2 raters), Krippendorff's alpha (≥3 or missing data). Raw percent agreement overstates reliability on imbalanced tasks — an 85% raw agreement at a 90% base rate is worse than it sounds. Rule of thumb: κ ≥ 0.7 to gate decisions. → `principles/human-evaluation.md`
- **Iteration leakage** — Overfitting an eval set through repeated engineer/agent decisions selected for raising its score — no training involved; "the engineers were the gradient." → `principles/contamination-and-leakage.md` §3
- **LLM-as-judge** — Using a model to score model outputs; an annotator you hired without an interview. Requires calibration, bias probes (position, verbosity, self-preference), and config pinning. → `principles/llm-as-judge.md`
- **McNemar's test** — Paired significance test on pass/fail flips between two systems on the same items; the discordant items are both the statistic and the triage worklist. → `principles/statistical-rigor.md`
- **Minimum detectable effect (MDE)** — The smallest true difference an eval can reliably distinguish from noise at its sample size (n ≈ 16·p(1−p)/Δ²). A suite promising sensitivity below its MDE causes random-walk engineering. → `principles/statistical-rigor.md`
- **Multiple-comparisons problem** — Slicing results into many metrics guarantees false "findings" at a predictable rate (~1 per 20 uncorrected tests); corrected with Benjamini-Hochberg/Bonferroni or by pre-registering few primary metrics. → `principles/statistical-rigor.md` §3
- **Position bias** — Pairwise judge favoring a slot regardless of content; detected by A/A testing, fixed by swap-and-require-consistency. → `principles/llm-as-judge.md` §2
- **Regression suite** — The eval tiers wired into CI that block changes on significant score drops; trustworthy only with measured noise bands, pinned baselines, and versioned instruments. → `principles/regression-testing-and-edd.md`
- **Reward model (RM)** — Model distilled from preference data to score outputs during RLHF-style training; inherits annotator and judge biases, which optimization then amplifies (reward hacking). → `topics/rlhf-preference-data-evals.md`
- **Rubric** — The written success criteria a scorer applies. Ambiguity here becomes arbitrary-but-consistent scorer house rules — the failure that looks like a working eval. → `principles/eval-design.md`, skill `eval-rubric-reviewer`
- **Self-preference bias** — Judge favoring outputs styled like its own model family; detected by cross-family re-judging, mitigated by third-family judges or panels. → `principles/llm-as-judge.md` §1
- **Shuffle test** — Running a suite under two item orderings; divergence beyond the noise band proves order-dependence (shared state, rate limiting, index drift). → `principles/cost-and-scalability.md` §3
- **Stratum / stratification** — Tagged subpopulations of a golden set (by intent, difficulty, modality, adversariality) reported separately; blended toplines hide per-stratum disasters and Simpson's-paradox mix effects. → `principles/eval-design.md`, `principles/statistical-rigor.md` §5
- **Trajectory(-level) evaluation** — Scoring an agent's full action sequence (tool calls, recoveries, side effects) against environment-verified outcomes, not just its final message. "Check the world, not the claim." → `topics/agentic-task-evals.md`
- **Traffic-weight audit** — Comparing each production intent/segment's share of the golden set vs. its share of real traffic; the detector for evals faithfully mapping territory nobody lives in. → `principles/production-offline-gap.md` §1
- **Verbosity bias** — Judge (or human) scoring longer outputs higher independent of content; detected by length–score correlation and content-identical compression probes. → `principles/llm-as-judge.md` §3
- **Verdict artifact** — Structured, immutable output file (numbers, CIs, hashes, exemplars) a subagent hands back; the antidote to verdict laundering across agent hand-offs. → `principles/multi-agent-orchestration.md` §4
- **Wilson interval** — CI method for proportions that behaves near 0%/100% (where safety evals live); the default for pass rates. → `principles/statistical-rigor.md`
