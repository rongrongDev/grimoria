# Glossary

Single source of terminology for the whole KB. One definition per term; docs link here instead of redefining. Terms are defined the way this KB *uses* them — where industry usage is sloppy (e.g. "exactly-once"), the entry says so.

- **At-least-once delivery** — messaging guarantee where every message is delivered one *or more* times; duplicates are normal, not exceptional. The default truth of every real broker. Consumers must be idempotent. See [concurrency.md](principles/concurrency.md) §3.
- **At-most-once delivery** — delivery zero or one times; loss is possible. Acceptable only for data you can afford to drop (metrics, telemetry).
- **Backfill** — populating existing rows with new/derived data after a schema change. Must be batched, resumable, idempotent, and run outside the deploy-time migration. [data-layer.md](principles/data-layer.md) §1.
- **Backpressure** — a system's ability to push resistance upstream when overloaded (bounded queues, 429s, shed load) instead of absorbing work it can't finish. Absence turns overload into latency and collapse. [async-work.md](principles/async-work.md) §2.
- **BOLA (Broken Object-Level Authorization)** — endpoint checks *that* you're authenticated but not that the object is *yours*; a.k.a. IDOR. The most common real-world API vulnerability. [security.md](principles/security.md) §1.
- **Bulkhead** — isolating resources (pools, threads, concurrency caps) per dependency so one slow dependency can't drain the shared pool and sink unrelated work. [observability.md](principles/observability.md) §4.
- **Cache stampede** — many concurrent requests recompute the same expensive value after a hot cache key expires, overwhelming the origin. Defenses: single-flight, stale-while-revalidate, TTL jitter. [concurrency.md](principles/concurrency.md) §6.
- **Circuit breaker** — wrapper that stops calling a failing dependency after a threshold (open), periodically trials recovery (half-open), and otherwise fails fast — protecting both caller and dependency. [observability.md](principles/observability.md) §4.
- **Contract test** — test verifying the *seam* between separately deployed services (consumer's expected requests/responses replayed against the provider), catching drift without a shared environment. [testing.md](principles/testing.md) §3.
- **Dead-letter queue (DLQ)** — destination for messages that exhausted retries or failed terminally. Requires an alert and a redrive path or it's silent data loss. [async-work.md](principles/async-work.md) §4.
- **Deadline propagation** — passing the caller's remaining time budget downstream so inner services stop working when the caller has given up. Native in gRPC; a header convention in HTTP. [observability.md](principles/observability.md) §2.
- **Dual write** — writing to two systems (DB + broker, DB + cache) without a transaction spanning both; a crash between the writes leaves them permanently inconsistent. Fixed by the outbox pattern. [concurrency.md](principles/concurrency.md) §5.
- **Error budget** — the allowed unreliability implied by an SLO (100% − target). Spend it shipping; when it's burned, reliability work wins by prior agreement. [observability.md](principles/observability.md) §3.
- **Eventual consistency** — replicas/derived systems converge to the source of truth after a delay, during which reads may be stale. An application-correctness concern (read-your-writes), not just an ops metric. [performance.md](principles/performance.md) §6.
- **Exactly-once processing** — the achievable form of "exactly-once": at-least-once delivery + idempotent consumers, so duplicates have no additional effect. Exactly-once *delivery* across a network does not exist. [concurrency.md](principles/concurrency.md) §3.
- **Expand/contract** — migration pattern: add the new shape (expand), dual-write and backfill and switch readers (migrate), remove the old shape (contract) — each phase a separate deploy, so old and new code always run against a compatible schema. [data-layer.md](principles/data-layer.md) §1.
- **Fencing token** — monotonically increasing number issued with a lock; the protected resource rejects writes bearing a stale token, closing the expired-lock/two-holders hole that TTL locks cannot. [concurrency.md](principles/concurrency.md) §2.
- **Graceful degradation** — pre-designed reduced-functionality modes under partial failure (fallbacks, feature shedding) instead of total failure. [observability.md](principles/observability.md) §4.
- **Head-of-line blocking** — one slow/stuck item delaying everything queued behind it (poison message at a queue head, slow message in a FIFO group, waiting DDL in a lock queue).
- **Hot key / hot partition / hot row** — one key/partition/row receiving disproportionate traffic, serializing a nominally parallel system at that point. [concurrency.md](principles/concurrency.md) §1, [stacks/mongodb.md](stacks/mongodb.md) §4.
- **Idempotency** — property that processing the same logical operation N times has the effect of exactly once. The backbone of safe retries. [concurrency.md](principles/concurrency.md) §4.
- **Idempotency key** — client-supplied unique identifier for a logical operation, letting the server detect and deduplicate retries; stored transactionally with the effect. [api-design.md](principles/api-design.md) §4.
- **Keyset (cursor) pagination** — paginating by "rows after this (sort-key, id) tuple" rather than by offset; stable under concurrent writes and O(page) instead of O(offset). [api-design.md](principles/api-design.md) §5.
- **Lock timeout** (`lock_timeout`) — cap on how long a statement waits to *acquire* a lock; mandatory before DDL so a blocked ALTER fails fast instead of queueing the world behind it. [data-layer.md](principles/data-layer.md) §1.
- **Lost update** — two read-modify-write cycles interleave and one write silently overwrites the other. Not prevented by READ COMMITTED. [data-layer.md](principles/data-layer.md) §2.
- **Mass assignment** — binding a raw request body onto a persistence model, letting attackers set fields they shouldn't (`role: admin`). Fixed by explicit input DTO allowlists. [security.md](principles/security.md) §6.
- **Mutation testing** — seeding artificial bugs (mutants) and checking tests fail; measures whether tests *verify*, not merely execute. [testing.md](principles/testing.md) §4.
- **N+1 query** — loading a list (1 query) then a per-item child query (N more); an ORM-manufactured performance defect. [data-layer.md](principles/data-layer.md) §3.
- **Optimistic locking** — detect conflicts at write time via a version column (`WHERE version = ?`); cheap when contention is rare, requires a handled retry/conflict path. [concurrency.md](principles/concurrency.md) §1.
- **Outbox pattern** — writing events to an `outbox` table in the same transaction as the business change; a relay publishes them afterwards. The standard fix for dual writes. [concurrency.md](principles/concurrency.md) §5.
- **Pessimistic locking** — taking the lock before reading (`SELECT ... FOR UPDATE`) so conflicting writers queue; right when contention is expected. [concurrency.md](principles/concurrency.md) §1.
- **Poison message** — message that fails deterministically on every delivery, blocking or looping forever without max-attempts + DLQ handling. [async-work.md](principles/async-work.md) §4.
- **Read-your-writes** — consistency guarantee that a client sees its own committed writes; broken by reading lagging replicas right after writing. [performance.md](principles/performance.md) §6.
- **Serializable / serialization failure** — strongest isolation level: outcomes equivalent to some serial order, enforced by aborting conflicting transactions (`40001`), which the application *must* retry. [data-layer.md](principles/data-layer.md) §2.
- **Single-flight (request coalescing)** — ensuring one concurrent computation per key while duplicates wait or take stale data; the in-process stampede defense. [concurrency.md](principles/concurrency.md) §6.
- **SLO (Service Level Objective)** — target for a user-journey reliability metric over a window ("99.9% of checkouts succeed < 500ms / 28 days"); the basis for error budgets and burn-rate alerting. [observability.md](principles/observability.md) §3.
- **SSRF (Server-Side Request Forgery)** — inducing the server to fetch attacker-chosen URLs, typically to reach internal networks or cloud metadata endpoints. [security.md](principles/security.md) §3.
- **Thundering herd** — many clients/processes acting in synchrony (reconnects, cache expiry, cron at :00) and overwhelming a shared resource; defused with jitter and coalescing. [concurrency.md](principles/concurrency.md) §6.
- **Visibility timeout** — period a received-but-unacked message is hidden from other consumers; shorter than processing time ⇒ systematic duplicate processing. [async-work.md](principles/async-work.md) §3, [stacks/messaging.md](stacks/messaging.md).
- **Write skew** — two transactions each read a shared invariant, both decide their write is safe, and the combined result violates it (e.g. both book the last slot). Requires SERIALIZABLE or explicit locking. [data-layer.md](principles/data-layer.md) §2.
